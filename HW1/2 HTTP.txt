   2. Задание № 2

 Что такое DNS (Domain Name System) и в чём разница протоколов HTTP/HTTPS и их версий - краткое эссе, своими словами

  DNS, это такая технология позволяющая в первую очередь пользоваться интернетом, а помогает она нам, направляя нас на нужный адрес сайта.
Например, когда мы вводим в поисковой строке браузера наш запрос (корректный), хотя если не корректный, современные технологии исправят
твой запрос на правильный (чаще), то наш браузер отправляет запрос на специальный сервер (DNS) для того чтоб узнать адрес (IP) и отобразить
содержимое которое мы хотим увидеть.
  Самое главное отличие протоколов HTTP от HTTPS, это буква S. Она обозначает безопасность данного сайта и шифрование личных данных, которые
потом расшифровываются при помощи специальных ключей на сервере. Можно не волноваться и смело передавать данные, что в HTTP будет опасно это делать, 
и должна всплыть информация о небезопасном подключении. Так же на безопасный протокол тратится больше ресурсов, и сайт может грузится 
незначительно дольше. 
  
HTTP / 1.0 - 1996 г.

В отличие от HTTP / 0.9, спроектированного только для HTML-ответов, HTTP / 1.0 справляется
и с другими форматами: изображения, видео, текст и другие изображения контента. В него добавлены
новые методы (такие, как POST и HEAD). Изменился формат запросов / ответов. К запросам и
ответам добавились HTTP-заголовки. Добавлены коды состояний, чтобы различать разные ответы
сервера. Введена поддержка кодировок. Добавлены составные типы данных (составные типы),
авторизация, кэширование, различные кодировки контента и ещё многое другое.
Вот так выглядели простые запрос и ответ по протоколу HTTP / 1.0:
GET / HTTP / 1.0
Хост: kamranahmed.info
Пользовательский агент: Mozilla / 5.0 (Macintosh; Intel Mac OS X 10_10_5)
Принимать: */*
